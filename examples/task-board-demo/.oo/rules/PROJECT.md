# Demo Workspace Rules

- Keep the example dependency-free; use Node.js built-ins only.
- Keep business logic in `src/task-board.mjs` and regression checks in `test/task-board.test.mjs`.
- Prefer small, reviewable changes that are easy to explain in a product demo.
- When behavior changes, update both tests and README notes if the user-facing flow changes.
