# @oneworks/plugin-chrome-driver 0.1.0-beta.5

- Added an “External Browser” settings subpage backed by an explicitly paired Manifest V3 bridge for controlling Chrome windows, tabs, groups, frames, browser data, page semantics, bounded debugging, and privacy settings with typed tools.
- Added exact target identities, per-target serialization, cross-target concurrency, resumable workflows, progressive step results, redacted audit summaries, and server-enforced risk confirmations.
- Added a trusted-origin oneWorks Web handshake and frame inventory with nonce, protocol negotiation, lifecycle cleanup, optional permission recovery, reconnect handling, and privileged/base extension flavors.
- Added default-off browser-session switches for Raw CDP/JavaScript, complete cookie values, and sensitive page fields. Raw is disclosed as browser-wide and a superset of the other switches, runs globally exclusively, blocks host-file escape primitives, and retains a target/origin navigation guard plus server-enforced per-use R4 confirmation. Pending review uses a redacted preview; persistent audit uses typed scopes and hashes.
- Added the shared 28px oneWorks action cursor, which moves smoothly and remains visible across a connected control session, plus a temporary Agent-cursor favicon on the exact Chrome tab during semantic click, type, select, key, and scroll operations; disconnect restores the page cursor state and each action completion or cancellation restores the latest favicon.
- Added isolated real-Chrome validation for permission recovery, reconnect and version mismatch, multi-window and multi-tab control, semantic interaction, cross-origin iframe isolation, and bookmark create/verify/confirm/cleanup.
- Added deterministic full developer and minimal Chrome ZIP packages with branded extension icons, manifest/version validation, checksums, GitHub build provenance, tag-driven releases, and an explicitly gated Chrome Web Store V2 publishing path for the developer package with `debugger` / `proxy` capability.
