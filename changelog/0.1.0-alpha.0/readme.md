# One Works 0.1.0-alpha.0

- Initialize the One Works app monorepo under `oneworks-ai/app`.
- Start the public package line from `0.1.0-alpha.0`.
- Use `oneworks`, `ow`, and `owo` as the CLI entrypoints.
- Move project runtime assets to `.oo` and cache data to `.oneworks`.
- Make local dev packaged runtime caches use the current build across desktop client/server/adapters, bootstrap runtime installs, and Android packaged metadata so installed dev apps do not fall back to stale published code.
